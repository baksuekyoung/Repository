보너스 과제 또 있는데 멀티모달 확장(업무 결과의 시각화)를 하래 생성된 결과(보고서/기획/요약)을 바탕으로 이미지 생성 AI를 활용해 1장 이상의 시각 자료를 만들어 보라는데 해 줄 수 있을까?

가능합니다. 다만 지금은 이미지 생성 한도에 도달해서 바로 만들 수는 없습니다.  
한도만 풀리면 **보고서/기획/요약 내용을 바탕으로 1장 이상의 시각 자료**를 만들 수 있고, 현재 단계에서는 바로 쓸 수 있는 **이미지 생성용 완성 프롬프트**를 만들어 두는 방식이 가장 좋습니다. [mywritingtwin](https://www.mywritingtwin.com/blog/custom-gpt-instructions-complete-guide)

## 바로 쓸 프롬프트

아래 문장을 이미지 생성 AI에 그대로 넣으면 됩니다.

```text
작은도서관 사무자동화 보고서를 바탕으로 만든 인포그래픽을 세련되고 실무적으로 표현해 주세요. 화면은 세로형 9:16 비율입니다. 상단에는 제목으로 '작은도서관 업무 자동화 결과'를 넣고, 중앙에는 네 개의 카드형 섹션을 배치합니다. 첫 번째 카드에는 일일 업무일지, 두 번째 카드에는 주간 업무일지, 세 번째 카드에는 도서 대출/반납 요약, 네 번째 카드에는 프로그램 계획서를 상징하는 아이콘과 간단한 수치를 표시합니다. 하단에는 '반복 업무 자동화로 시간 절감과 문서 품질 향상'이라는 문구를 넣습니다. 전체 분위기는 깔끔하고 신뢰감 있는 행정 보고용 디자인으로, 흰색 바탕에 파란색과 연회색 포인트를 사용하고, 아이콘과 도형은 단순하고 명확하게 표현합니다. 텍스트는 정확히 읽히도록 선명하게 배치해 주세요.
```

## 추천 구성

시각 자료는 보통 아래 3가지 중 하나로 만들면 좋습니다.

- **인포그래픽 1장**: 핵심 결과를 한눈에 보여줌.
- **비교표 스타일 카드 이미지**: 모델 비교, 점수표, 장단점 정리용.
- **흐름도 이미지**: 입력 → 처리 → 출력 → 검수 순서를 보여줌.

작은도서관 과제라면 **인포그래픽 1장 + 흐름도 1장** 조합이 가장 설득력 있습니다. [agentpatterns](https://agentpatterns.ai/instructions/prompt-file-libraries/)

## 추천 시각화 주제

- 작은도서관 업무 자동화 성과 요약.
- 프롬프트 엔지니어링 작업 흐름.
- 4축 모델 비교 결과.
- PASS/FAIL 검증 흐름.
- 버전관리와 롤백 절차.

이런 이미지는 보고서의 내용을 시각적으로 보완해 주고, 동료평가나 사전평가에서 “정리 능력”과 “전달력”을 보여주기에 좋습니다. [developersdigest](https://www.developersdigest.tech/blog/gpt-image-2-prompt-library-production)
